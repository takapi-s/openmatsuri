# OpenMATSURI 製品ロードマップ

Admin / Viewer / Tracker の Web 版を中心とした機能開発計画。ネイティブアプリ展開は [native-roadmap.md](./native-roadmap.md) を参照。

## 方針

- 祭りごとの**見た目のカスタマイズ**（アイコン・ブランディング）で運営の満足度を上げる。
- 来場者・運営双方からの**フィードバック**を収集し、改善サイクルを回せるようにする。
- 既存スキーマ（`trackers.icon_url` / `icon_color` など）を活かしつつ、UI と Viewer 表示まで一気通貫で仕上げる。

---

## Phase 1: マップ表示のカスタマイズ

### トラッカーアイコン

山車・神輿・だんじりなど、トラッカーごとにマップ上の見た目を変える。

- [ ] Admin でトラッカーごとにアイコンを設定できる UI（色・アイコン画像の選択またはアップロード）
- [ ] `trackers.icon_url` / `icon_color` の編集・保存（現状は DB カラムのみ、管理画面未対応）
- [ ] Viewer マップでカスタムアイコンを表示（画像アイコン + 色のフォールバック）
- [ ] アイコン画像の Storage 保存（Supabase Storage）とサイズ・形式のガイドライン

### イベントごとのアイコン

イベント単位で Viewer / 埋め込みマップのブランディングを統一する。

- [ ] `events` にアイコン・ロゴ用フィールドを追加（例: `icon_url`, `brand_color`）
- [ ] Admin のイベント設定でアイコン・テーマカラーを編集
- [ ] Viewer のヘッダー・OGP・埋め込み iframe でイベントアイコンを表示
- [ ] トラッカーのデフォルトアイコン色をイベント設定から継承するオプション

---

## Phase 2: フィードバック

来場者・運営からの意見をアプリ内で受け付ける。

- [ ] Viewer に「フィードバックを送る」導線（簡易フォーム or 外部リンク）
- [ ] Admin でイベントごとのフィードバック一覧を確認
- [ ] フィードバック用テーブル（`event_id`, 本文, 種別, 任意の連絡先, 作成日時）
- [ ] スパム対策（レート制限、honeypot、必要に応じて CAPTCHA）
- [ ] 運営向け通知（メール / Webhook は将来検討）

---

## Phase 3: 運営・コラボレーション

- [ ] 実行委員会メンバーの招待（メールアドレス）— **実装済み**
- [ ] 未登録ユーザー向けの招待リンク（トークン付き URL）
- [ ] メンバーの権限変更・削除（オーナー操作）
- [ ] 監査ログ（誰がいつ設定を変更したか）

---

## Phase 4: その他（候補）

優先度は Issue / コミュニティの要望で調整する。

- [ ] イベント複製（設定・POI・コースのテンプレート化）
- [ ] 公開スケジュール（`starts_at` / `ends_at` と Viewer 表示の連携強化）
- [ ] 多言語対応（Viewer / Admin）
- [ ] アクセス解析ダッシュボード（来場者数・ヒートマップの期間集計）
- [ ] 公式テンプレート（よくある祭り構成のワンクリックセットアップ）
